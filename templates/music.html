Music

Background

Notes

A simple way to think of a song is as a sequence of notes. In music, each note is assigned a letter from A through G, inclusive. These notes correspond to the white keys (otherwise known as the "natural keys") on a piano keyboard, as shown in the image below.

Notice how each letter actually represents multiple possible notes. C, for instance, appears both at the beginning of this sequence of keys and just after the first B. In music theory, we say that these notes which have the same letter but represent different keys are members of the same "pitch class," but are in different "octaves." (Note that the word "octave" is related to the fact that there are eight natural keys between the two Cs, including both endpoints.) Thus, the two Cs will sound similar in quality, but the second one, as a member of a higher octave, will have a higher pitch!

Now, how do we refer to the black keys on a piano keyboard? To do that, we need to use two "accidentals": symbols which indicate that a note should be moved to the immediate next or immediate prior note. Appending a letter with the "sharp" accidental (♯), often represented in ASCII using the # symbol, means the note should be moved up one semitone (i.e., a single piano key), while appending a letter with the "flat" accidental (♭), often represented in ASCII with b, means the note should be moved down one semitone.

This gives us the picture below.

You may have noticed that this notation allows for multiple different ways to represent the same note. F♯, the note we get by starting at F and moving up one semitone, is the same as G♭, the note we get by starting at G and moving down one semitone. And, though not shown here, we could likewise represent the note F as E♯ or the note B as C♭.

Sheet Music

When a song is written out in "sheet music," the song is divided into "measures" which are separated by vertical bars.

The above piece of music has three measures. The 4/4 at the beginning of the piece is just a convention which means that each measure is divided into enough time for four "quarter notes" (i.e., a note that lasts a quarter of a measure.) Which note should be played depends upon the line or space between lines the note is drawn on. Each line or space between lines represents one of the natural keys on the piano, with notes on higher lines representing higher pitched notes. The letters correspond to each note are included here, though typically the letters themselves are omitted from the sheet music in practice.

Notice again how there can be notes represented by the same letter, but in different octaves: each of C, D, E, F, and G appear twice in different octaves in the sheet music above. To be more precise about which note we wish to refer to, sometimes musicians will place an octave number after the letter to represent the note’s octave as well as pitch.

Sheet music also needs a way to represent accidentals, like the ♯ and ♭ symbols. For instance, to indicate the note G♯ in sheet music, musicians place a note on the line assigned to G, and then add a ♯ symbol before it. The same can be done for ♭.

What about representing a pause between notes? In music, a pause between notes is called a "rest." There are different symbols for different rests, depending on how long they are.

But what if we don’t want all of our notes to have the same length? Well, just as a quarter note is a note which lasts a quarter of the measure, an eighth note is a note which lasts an eighth of a measure (assuming 4/4 time). Two eighth notes are typically represented as two notes connected by a bar atop both of them, like the below.

In this example, the first two notes (both C4) are eighth notes which each take up an eighth of the measure. The next four notes (two G4s followed by two A4s) are also eighth notes. That’s a total of six eighth notes, which together take up 6/8 = 3/4 of the time in this measure. The remaining 1/4 of the time in this measure is taken by the final G4, a quarter note, which lasts twice as long as the other notes in this measure.

MIDI

Okay, now back to computers. A common way to store music on a computer is as a MIDI (i.e., Musical Instrument Digital Interface) file. A MIDI file consists of a 14-byte "header chunk," which includes instructions about the file’s format and the number of tracks in the music file. With multiple tracks, you could create a MIDI file that has multiple sounds playing simultaneously! For now, though, we’ll stick to MIDI files with one track, which means that only one note will play at a time.

After the header chunk comes the "track chunk," which generally consists of a sequence of MIDI "events." For note events, each event is represented by:

  • time delta (usually 1 byte, but can be more): the amount of time between the prior event and this event.

  • event type (4 bits): what sort of event should happen. According to the MIDI standard, event type 0x8 (or 1000 in binary) is the event type for beginning a note, and event type 0x9 (1001 in binary) is the event type for ending a note.

  • event channel (4 bits): which "channel" the event should happen on. Playing on different channels allows multiple sounds to be played simultaneously.

  • note number (1 byte): which note to be turned on or off. Each note is represented by a unique number in the MIDI standard.

  • velocity (1 byte): the speed at which the note should be hit, which roughly corresponds to volume.

Take a look at midi.h. In it, you’ll see some structures that we’ve defined for representing MIDI files and events, as well as some functions we’ve declared for interacting with MIDI files. Read through the comments and function signatures for create_song, add_note, add_rest, and write_song to get a sense for how they work.

Okay, now let’s see these functions in action. Open up scale.c and look at the main function. You’ll see that this program first creates a song, then adds a sequence of notes to the song (representing the "C scale"), and then writes the song to a MIDI file called scale.mid. Go ahead and run the below.

$ make scale

Then execute the program via the below.

$ ./scale

A file called scale.mid should appear in your current directory! Download it and listen to it. If you have trouble opening the file, you may want to download QuickTime 7. You should be able to hear the music!

Representing Notes

Now, how do we represent sheet music using just ASCII symbols? Let’s have every three characters stand for an eighth note. To represent the note G♯ in the fourth octave as an eighth note, we would thus write G#4. We could likewise represent the same note but flat as Gb4, and the natural version of the note as G4_ (notice that we add an _ at the end of the note to maintain a three-character representation of eighth notes.)

What about notes that last more than an eighth of a measure? We can let the sequence ... indicate that a note should continue for an extra eighth note. So the notation G5_... would represent the key G (natural) in the fifth octave, lasting one quarter note (two eighth notes). And the notation D#4......... would represent the key D♯ in the fourth octave, lasting four eighth notes (half a measure).

Finally, we can represent a rest lasting one eighth of a measure as a sequence of three spaces . You can see some examples of representing notes and this rests in this way in the songs/ directory. Each file represents a song, with all of the notes and rests represented in ASCII on a single line.

Specification

Implement a program called music that generates MIDI files from a sequence of notes.

  • Implement your program in a file called music.c in a directory called music.

  • Your program should accept exactly one command-line argument, the name of the MIDI file which your program will eventually generate.

    • If your program is not executed with exactly one command-line argument, it should remind the user of correct usage, as with fprintf (to stderr), and main should return 1.

  • Your program should first get (via stdin) a string from the user, which is the ASCII representation of the song you should generate, formatted according to the above specification.

    • This means that you can also pass in the contents of a file as the notes to generate via file redirection (remember how?)

  • Your program should then output a playable MIDI audio file based on those notes.

  • If your program is passed an invalid representation of a song, it should display an error message, and main should return 2.

Usage

Your program should behave per the examples below. Assumed that the underlined text is what some user has typed.

$ ./music
Usage: ./music output.mid
$ ./music bday.mid < songs/bday.txt
$ echo $?
0
$ ./music output.mid
D4_...F#4...A4_...D5_...   D5_...A4_...F#4...D4_...
$ echo $?
0
